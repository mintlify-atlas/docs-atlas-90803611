## Protocol Buffers

All services are defined using Protocol Buffers v3. The proto files are located in `modules/meteroid/proto/api/`.

## Authentication

All gRPC endpoints require authentication. Refer to the authentication documentation for details on how to authenticate your requests.

## Error Handling

The API uses standard gRPC status codes for error responses:

- `OK` (0): Success
- `INVALID_ARGUMENT` (3): Invalid request parameters
- `NOT_FOUND` (5): Resource not found
- `ALREADY_EXISTS` (6): Resource already exists
- `PERMISSION_DENIED` (7): Insufficient permissions
- `INTERNAL` (13): Internal server error

## Pagination

Many list endpoints support pagination using the `meteroid.common.v1.Pagination` message:

```protobuf
message Pagination {
  uint32 limit = 1;
  uint32 offset = 2;
}
```

Responses include a `PaginationResponse` with metadata about the total count and current page.
