## Overview

The Organizations service provides organization-level management:

- List and retrieve organizations
- Create new organizations
- Access organization and tenant information

## Service Definition

```protobuf
service OrganizationsService {
  rpc ListOrganizations(ListOrganizationsRequest) returns (ListOrganizationsResponse) {}
  rpc GetCurrentOrganizations(GetCurrentOrganizationRequest) returns (GetCurrentOrganizationResponse) {}
  rpc CreateOrganization(CreateOrganizationRequest) returns (CreateOrganizationResponse) {}
}
```

## RPC Methods

### ListOrganizations

Retrieves all organizations accessible to the authenticated user.

**Request:** Empty request (no parameters)

**Response:** Returns list of `Organization` objects.

### GetCurrentOrganizations

Retrieves the current organization context with tenant details.

**Request:** Empty request (no parameters)

**Response:** Returns `OrganizationWithTenant` including associated tenants.

## Organization vs Tenant

Meteroid uses a multi-tenant architecture:

- **Organization**: Top-level entity representing a company or business unit
- **Tenant**: Isolated environment within an organization (e.g., production, staging, different business lines)

Each organization can have multiple tenants, allowing for:
- Environment separation (prod/staging/dev)
- Multi-brand management
- Department isolation
